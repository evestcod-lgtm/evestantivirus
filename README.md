# Evest Antivirus

Android-антивирус на Kotlin + Jetpack Compose. Реальное сканирование установленных
приложений и APK-файлов, журнал угроз и журнал защиты, настраиваемый автоскан,
уведомления, честная логика карантина/удаления в рамках того, что разрешает Android
без root.

## Что реально делает приложение

- Сканирует список установленных приложений через `PackageManager` и сверяет
  пакетные имена с локальной базой сигнатур (`app/src/main/assets/threat_db/signatures.json`).
- Анализирует комбинации запрошенных разрешений по эвристическим паттернам
  (например: спец.возможности + оверлей + SMS — характерно для банковских троянов).
- Считает SHA-256 хэши APK-файлов в папке Загрузки и сверяет с базой хэшей.
- Опционально может проверять хэш файла через публичный VirusTotal API,
  если ты укажешь свой бесплатный API-ключ в настройках (без ключа функция просто не используется).
- Ведёт два журнала в локальной базе Room: журнал угроз и журнал событий защиты.
- Присылает уведомления о результатах сканирования, найденных угрозах и очистке.
- Автоматическая проверка через `WorkManager` — интервал настраивается в приложении,
  можно полностью отключить. Минимум — 15 минут (это ограничение самого Android,
  WorkManager не запускает периодические задачи чаще).
- Честно предлагает удалить приложение через системный диалог (Android не разрешает
  стороннему приложению удалить другое без подтверждения пользователя) и умеет
  реально переносить подозрительные файлы в приватный карантин приложения.

## Структура проекта

```
EvestAntivirus/
├── app/src/main/java/com/evest/antivirus/
│   ├── scan/          — движок сканирования (AppScanner, FileScanner, ScanEngine)
│   ├── threatdb/       — модель и загрузчик базы сигнатур, обновление по URL
│   ├── data/           — Room (журналы), DataStore (настройки)
│   ├── notifications/  — уведомления, WorkManager автоскан, BootReceiver
│   ├── quarantine/     — карантин файлов и запрос удаления приложений
│   ├── sounds/         — воспроизведение звуковых эффектов
│   ├── battery/        — запрос исключения из оптимизации батареи
│   ├── permissions/    — список разрешений с честными объяснениями
│   └── ui/             — Compose экраны, тема, навигация, ViewModel'и
├── app/src/main/assets/threat_db/signatures.json   — пример базы угроз
├── app/src/main/res/raw/           — сюда кладёшь свои mp3 и видео
├── app/src/main/res/drawable/avatar_evesteven.*     — аватар-талисман
└── .github/workflows/build-apk.yml — автосборка APK на GitHub
```

## Твои файлы — куда класть

Смотри `assets_for_user/sounds/README.md`, `assets_for_user/video/README.md`,

перемещать звук, видео и фото, чтобы Android их подхватил.

## Как собрать APK через GitHub (пошагово, без Android Studio)

### Шаг 1. Создай репозиторий на GitHub
1. Зайди на github.com → New repository.
2. Назови его, например, `evest-antivirus`. Публичный или приватный — не важно.
3. **Не** добавляй README/gitignore при создании — они уже есть в проекте.

### Шаг 2. Загрузи проект в репозиторий
Самый простой способ — прямо в браузере:
1. Открой созданный репозиторий на GitHub.
2. Нажми **Add file → Upload files**.
3. Перетащи туда всю папку `EvestAntivirus` (или содержимое zip-архива, который я отдал).
4. Нажми **Commit changes**.

Если удобнее через терминал (компьютер):
```bash
cd EvestAntivirus
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ_НИК/evest-antivirus.git
git push -u origin main
```

### Шаг 3. Собери APK автоматически через GitHub Actions
1. Открой вкладку **Actions** в репозитории на GitHub.
2. Если появится предложение включить Actions — нажми **I understand my workflows, go ahead and enable them**.
3. Найди workflow **Build Evest Antivirus APK** слева.
4. Если сборка не запустилась сама после пуша — нажми **Run workflow** (кнопка справа) → **Run workflow**.
5. Подожди 3–7 минут, пока идёт сборка (статус — жёлтый кружок, потом зелёная галочка).

### Шаг 4. Скачай готовый APK
1. Открой завершённый запуск workflow (зелёная галочка).
2. Внизу страницы — раздел **Artifacts**.
3. Скачай **evest-antivirus-debug-apk** (это готовый к установке файл, уже подписан отладочным ключом Android).
4. Внутри будет `.apk` — скачай его на телефон и установи (может понадобиться разрешить установку из неизвестных источников в настройках телефона).

> Файл `evest-antivirus-release-unsigned-apk` собирается тоже, но он **не подписан**
> и не установится напрямую — это заготовка для случая, если ты в будущем захочешь
> подписать релизную версию своим ключом. Для личного использования тебе нужен именно debug APK.

### Если сборка упадёт с ошибкой
Открой лог упавшего шага в Actions — там будет понятная ошибка Gradle/Kotlin.
Самые частые причины:
- Не удалён `avatar_evesteven.xml` при добавлении своего `avatar_evesteven.png` (см. инструкцию в assets_for_user/avatar).
- Файл звука/видео назван с заглавными буквами или пробелом — Android требует строчные буквы и `_`.

## Честные ограничения (важно понимать)

Android с версии 8+ **не позволяет** сторонним приложениям без root:
- удалять другие приложения без подтверждения пользователя в системном диалоге;
- читать файлы других приложений;
- останавливать/лечить код внутри чужого APK "изнутри".

Evest Antivirus работает строго в рамках официальных API Android: анализирует
метаданные (пакеты, разрешения, хэши файлов, источники установки) и предлагает
пользователю принять решение там, где решение обязано принимать он, а не приложение.
Это тот же принцип, на котором работают все антивирусы для Android без root.

## Расширение базы угроз

`app/src/main/assets/threat_db/signatures.json` — обычный JSON, можно редактировать
без пересборки кода: закинуть новый файл `signatures.json` того же формата на
свой GitHub (или любой хостинг) и указать ссылку в настройках приложения —
кнопка "Обновить базу" скачает и провалидирует новую версию.
